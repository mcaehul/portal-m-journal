# portal-m-journal

## 项目简介

`portal-m-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
它不针对任何特定域名或网站，而是作为一个通用的页面集合，供浏览、参考或引用。  
每个页面均为独立的静态文件，结构清晰，便于维护和分发。

## 目录说明

```
portal-m-journal/
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 页面引用的资源文件（样式、脚本、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，命名方式通常为 `page-xxx.html` 或 `xxx/index.html`。
- **assets/**：公共资源文件，按类型组织（如 `css/`、`js/`、`images/`）。
- **index.html**：仓库的入口页面，可用于展示页面列表或导航。

## 页面归档说明

所有页面均为独立的静态 HTML 文件，不依赖外部数据库或后端服务。  
归档遵循以下原则：

- 每个页面包含完整的 HTML 结构（`<html>`、`<head>`、`<body>`）。
- 页面内引用的资源优先使用相对路径，避免跨域问题。
- 页面之间无直接耦合，可单独打开或部署。

如需新增页面，直接在 `pages/` 目录下创建文件，并确保资源路径正确即可。

## 维护说明

- **更新页面**：直接编辑 `pages/` 下的对应文件，或替换为最新版本。
- **添加页面**：在 `pages/` 中新建文件，并在 `index.html`（如有）中添加链接。
- **删除页面**：移除对应文件及 `index.html` 中的相关引用。
- **资源管理**：新增或修改资源时，注意保持 `assets/` 目录的结构整洁。

本仓库不提供自动构建或发布脚本，所有更新需手动提交并推送至 GitHub。  
如需部署到静态托管服务（如 GitHub Pages），可将仓库根目录或 `pages/` 目录设为站点根目录。
